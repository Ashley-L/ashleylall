Superdaze: Web article

Superdaze was produced for my Interface Development II class in second year with my team member Kim Nguyen.

This final assignment took about a month to complete. The purpose of this project was to learn about how to improve the user experience of reading an article on a website. There were four major requirements: the implementation of a scrollspy to help the user understand where they are on the page, a carousel component, an infinite scroll, and one additional component that would improve user experience.

I am a huge fan of the Marvel Cinematic Universe, so I immediately suggested we do something superhero-related. Thus, the superhero website ‘SUPERDAZE’ was born, as was our article ‘Top 8 Marvel Heroes and Villains.’ The design of page was styled with clasically bright and gaudy superhero websites in mind.

Features

Scrollspy

After we wrote the content and finished our basic CSS, we moved on to the JavaScript component. I able to write code for each component we planned on making.

I was originally tasked with designing the scrollspy that would allow the user to navigate to different characters on the page. If the user’s scroll position was within the bounding rectangle of a character section, the little character head (which I made in Illustrator!) would increase in size to indicate that the user was currently viewing the respective character’s profile. One issue I have noticed with this is that there is a small strip of space after the hero section and before the villains section where there is no character content to read, leaving the scrollspy inactive. This makes it a bit difficult for the user to identify where they are on the page.

Carousel

Our carousel component would be clickable character thumbnails in each section. I was not initially tasked with coding this component, but I helped troubleshoot when my partner was having issues. We were having trouble with getting the big character image to reflect the clicked thumbnail. The solution we came up with was to change the image source of the big image to that of the clicked thumbnail and do this for each character section.

Infinite Scroll

To make the infinite scroll component, I had to create a new element when the user’s scroll position was near the bottom of the original article. To make the appearance of the new article less jarring, we added a fade transition.

Extra Features

For the additional UX components, we had decided on an option to filter characters by movie, which had seeming daunting at first, but I found the logic in the end to be rather simple. I applied class lists based on movie to every character, so that when I clicked on a movie tag, I would show all characters with the respective movie class and hide the content of those without it. This works, though one change I would make to the filtering system overall is to have the functionality to deselect a button and display all characters again. The work-around I added in was to select an ‘all’ button if the user wants to see all characters after filtering by movie.

Near the end of the project, I suggested adding a fixed footer for an additional level of usability. An infinite scroll was required, but I found it to make the site less usable – the user would never be able to reach the footer.

Next steps

This was an enjoyable project to work on, especially since I was so enthusiastic about the subject matter. I’ve outlined some bugs I’m currently aware of and I think those pose as a good starting point for improvement. I definitely learned a lot about some of the uses JavaScript can have in making a webpage interactive and I hope to continue to have opportunities to apply these skills to future projects.
